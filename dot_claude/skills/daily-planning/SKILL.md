# Daily Planning スキル

毎朝のプランニングを4ステップで実行し、Obsidianに今日の日報ファイルを生成する。

## 設定

- **Vaultパス**: `/Users/hakshu/Library/Mobile Documents/iCloud~md~obsidian/Documents`
- **日報フォルダ**: `{VAULT}/日報/`
- **タスクファイル**: `{VAULT}/タスク.md`
- **プロジェクトフォルダ**: `{VAULT}/プロジェクト/`

---

## セットアップ確認

初回実行時、`gws` コマンドが未インストールなら案内する：

```bash
# インストール確認
which gws

# 未インストールの場合
brew install googleworkspace/tap/gws
# または
go install github.com/googleworkspace/cli/cmd/gws@latest

# 初回認証
gws auth login
```

---

## 実行フロー

### ステップ1: コンディションチェック

ユーザーに今日の調子を聞く：

```
今日の調子はどう？

1. 好調
2. ふつう
3. ちょっと低め
```

入力を受け取ったらそのまま次のステップへ。コンディションに応じたコメントを一言添えてもいい（低めなら「無理せず行こう」など）。

---

### ステップ2: Googleカレンダー

今日の予定を取得して作業可能時間を算出する。

```bash
# 今日の日付（YYYY-MM-DD形式）
TODAY=$(date +%Y-%m-%d)
TOMORROW=$(date -v+1d +%Y-%m-%d 2>/dev/null || date -d tomorrow +%Y-%m-%d)

# 今日のイベント取得（RFC3339形式）
gws calendar events list \
  --params "{\"calendarId\": \"primary\", \"timeMin\": \"${TODAY}T00:00:00+09:00\", \"timeMax\": \"${TOMORROW}T00:00:00+09:00\", \"singleEvents\": true, \"orderBy\": \"startTime\"}"
```

取得したJSONから以下を抽出・表示する：
- 各イベントの開始時刻・終了時刻・タイトル
- MTGの合計時間（分）
- **作業可能時間** = 稼働時間（9:00〜18:00 = 9時間）- MTG合計 - 昼休憩1時間

表示フォーマット：
```
📅 今日のスケジュール

09:00-10:00  チームスタンドアップ
14:00-15:00  1on1

MTG合計: 2時間
作業可能時間: 6時間
```

`gws` が未インストール・未認証の場合はセットアップを促し、カレンダー部分をスキップして次のステップへ。

---

### ステップ3: Next Actions

タスクファイルとプロジェクトファイルを読み込んで、進行中タスクと期限を表示する。

```bash
VAULT="/Users/hakshu/Library/Mobile Documents/iCloud~md~obsidian/Documents"

# タスクファイルを読む
cat "${VAULT}/タスク.md"

# プロジェクトファイルがあれば読む
ls "${VAULT}/プロジェクト/" 2>/dev/null && \
  for f in "${VAULT}/プロジェクト/"*.md; do cat "$f"; done
```

読み込んだ内容から未完了タスク（`- [ ]`）を抽出し、期限（📅 YYYY-MM-DD）でソートして表示する。

期限が**今日以前のもの**は⚠️、**3日以内**は🔶、**それ以降**は通常表示。

表示フォーマット：
```
📋 Next Actions

【プロジェクトA】
  ⚠️  タスク名  (期限: 3/5 - 超過)
  🔶 タスク名  (期限: 3/9)

【プロジェクトB】
  タスク名  (期限: 3/20)
```

タスクがなければ「タスクファイルに何も書かれていないよ。タスク.mdにプロジェクトを追加してね」と伝える。

---

### ステップ4: 直近の日報チェック

直近5日分の日報から「明日に繰り越し」セクションの未完了タスクを拾う。

```bash
VAULT="/Users/hakshu/Library/Mobile Documents/iCloud~md~obsidian/Documents"
DAILY_DIR="${VAULT}/日報"

# 直近5ファイルを日付降順で取得
ls -t "${DAILY_DIR}"/*.md 2>/dev/null | head -5
```

各ファイルの「明日に繰り越し」セクションにある `- [ ]` 行を抽出して表示する。

```
📌 繰り越しタスク（直近の日報より）

2026-03-06 より:
  - タスク名

2026-03-05 より:
  - タスク名
```

日報がない場合はスキップ。

---

### ステップ5: 日報ファイルの生成

集めた情報を使って今日の日報ファイルを生成する。

ファイルパス: `{VAULT}/日報/{YYYY-MM-DD}.md`

コンディションの絵文字：
- 好調 → 🟢
- ふつう → 🟡
- ちょっと低め → 🔴

```markdown
---
date: 2026-03-07
condition: ふつう
---

# 2026-03-07

## コンディション
🟡 ふつう

## 今日のスケジュール

| 時間 | 予定 |
|------|------|
| 09:00-10:00 | チームスタンドアップ |
| 14:00-15:00 | 1on1 |

作業可能時間: **6時間**

## 今日やること

- [ ] （Next Actionsから期限順にピックアップ）
- [ ] （繰り越しタスク）

## メモ

## 振り返り

### できたこと

### 明日に繰り越し

```

ファイル生成後、Obsidianで開くコマンドを案内する：
```
open "obsidian://open?vault=Documents&file=日報%2F2026-03-07"
```

---

## エラーハンドリング

- **gws未インストール**: インストール手順を案内してカレンダー部分をスキップ
- **gws認証切れ**: `gws auth login` を実行するよう案内
- **タスクファイルなし**: スキップして「タスク.mdを作成してね」と伝える
- **日報フォルダなし**: `mkdir` で作成してから進む
- **今日の日報が既に存在**: 上書きせず「すでに存在するよ、開く？」と確認する
